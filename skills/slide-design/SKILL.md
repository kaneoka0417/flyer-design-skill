# プレゼンテーションスライドデザインスキル

スライドをデザインし、高画質PDFを生成するためのガイド。

## 絶対ルール（非交渉）

1. **1スライド1メッセージ** -- 複数のアイデアを1枚に詰め込まない。各スライドは1つの明確なメッセージだけを伝える
2. **絵文字禁止** -- テンプレート・成果物に絵文字を一切使わない。全てインラインSVGアイコンで置換する（参照: `references/icons-images.md`）
3. **テキスト最小限** -- タイトル+キーメッセージ+ビジュアルの3要素以内。スライドは口頭プレゼンの視覚補助であり、読ませる資料ではない
4. **フォントサイズ下限** -- タイトル40pt以上、本文18pt以上、キャプション14pt以上。プロジェクター投影ではさらに大きく
5. **余白は武器** -- 空白を恐れない。スライド面積の20%以上は空白にする。余白が情報の呼吸を作る
6. **高画質出力** -- PDF生成は必ず高画質設定（deviceScaleFactor:2, /prepress, 600dpi）

## ワークフロー

### Step 0: 要件ヒアリング（徹底）

以下の項目を**全て確認するまで**デザインに進まない。不足があれば追加質問を続ける。

**必須確認項目:**

| 項目 | 質問例 |
| --- | --- |
| **目的** | 何を伝えたいか（営業提案、セミナー、社内報告、イベント等） |
| **ターゲット** | 誰に届けるか（役職、年齢層、人数） |
| **持ち時間** | プレゼンの持ち時間 -> スライド枚数目安（1分/枚） |
| **テーマ** | ダーク or ライト（使用環境・ブランドに合わせて提案） |
| **ブランドカラー/ロゴ** | 既存のブランドカラーやロゴデータの有無 |
| **使用環境** | プロジェクター/大型モニター/PDF配布/Web会議 |
| **掲載情報** | タイトル、キーメッセージ、データ、事例、CTA等 |
| **写真の有無** | 使用したい写真があるか |
| **既存資料** | チラシ・企画書など転用できる既存素材の有無 |

### Step 0.5: テンプレート選択

要件に基づいて、テーマテンプレートを選択する。

| 条件 | テンプレート |
| --- | --- |
| ダーク | `assets/slide-templates/theme-dark.html` |
| ライト | `assets/slide-templates/theme-light.html` |

選択したテンプレートを読み込み、CSS変数でブランドカラーに調整する。

### Step 1: デザイン哲学の策定

レイアウトに入る前に、スライド全体の美的方向性を言語化する。

**策定内容:**

- **ムーブメント名**（1-2語）: デザインの核心を表現する造語（例: "Strategic Vision", "Clean Impact"）
- **ビジュアルトーン**: 写真主体 / アイコン主体 / テキスト主体
- **配色方針**: 3色の法則（背景70% / メイン25% / アクセント5%）に基づく具体的な色と感情的意図
- **タイポグラフィ方針**: フォントの組み合わせとその理由

**品質基準（非交渉）:**

- 1スライド1メッセージ -- 1枚に複数の概念を詰め込まない
- 視覚的インパクト -- 3秒でそのスライドのメッセージが伝わること
- 一貫性 -- 全スライドで統一されたデザイン言語

**出力**: `output/[project]/slide-design-philosophy.md` として保存

### Step 2: 構成案作成（スライドマップ）

デザイン哲学をもとに、全スライドの構成案を作成する。

**各スライドに含める内容:**

- **スライド番号**
- **スライドタイプ**（9種から選択）
- **タイトル文言**
- **キーメッセージ**（1文以内）
- **ビジュアル要件**（写真/アイコン/図表/なし）
- **スピーカーノート**（口頭で補足する内容）

**1スライド1メッセージの原則を厳守。** 情報が多い場合はスライドを分割する。

**スライド枚数の目安:**

| 持ち時間 | スライド枚数 |
| --- | --- |
| 5分 | 5-7枚 |
| 10分 | 10-12枚 |
| 15分 | 15-18枚 |
| 30分 | 25-35枚 |

**出力**: `output/[project]/slide-composition.md` として保存

### Step 3: 写真・素材の確定

構成案の完成後、実装に入る前に使用する写真・素材を確定する。

#### 3.1 ユーザーへの素材ヒアリング（必須）

構成案のビジュアル要件をもとに、ユーザーへ以下を確認する:

- **手持ち素材の有無**: 「構成案で◯箇所に写真を使用します。使用したい写真の候補はありますか？」
- **提供方法の案内**: ファイルパス指定、フォルダごと共有、画像の直接貼り付け等
- **素材がない場合の方針**: フリー素材を使うか、撮影するか、仮画像で進めるか

**ユーザーに確認せず、勝手にフリー素材やプレースホルダーで進めない。**

#### 3.2 写真候補の評価・選定

ユーザーから候補を受け取ったら、以下の観点で S/A/B/C の4段階評価を行う:

| 評価観点 | 内容 |
| --- | --- |
| **解像度** | スライド全面使用に耐えるか（1920x1080px以上 -> S/A、低解像度 -> C） |
| **構図・被写体** | 該当スライドの役割に合った内容か |
| **明るさ・色調** | デザイン哲学の配色方針と調和するか |
| **トリミング耐性** | `object-fit: cover` で16:9に切り抜いたとき、重要な被写体が残るか |

#### 3.3 配置マッピングと不足素材の対応

- 各候補を構成案のどのスライドに配置するか提案する
- 構成案の要件に対して候補が足りない場合、**具体的なシチュエーション・撮影指示**を提示する
- フリー素材で補う場合は `references/icons-images.md` の推奨サイトから選定する

**出力**: 選定結果を `output/[project]/slide-composition.md` のビジュアル要件に反映

### Step 4: Reveal.js + HTML/CSS実装

**実装前に必ず `references/slide-design-system.md` を読むこと。**
これがスライドデザインの核心ルール集であり、全てのデザイン判断の基盤。

**実装手順:**

1. Step 0.5で選択したテーマテンプレートを `output/[project]/` にコピー
2. CSS変数（`--accent`, `--bg-*`, `--text-*`）をブランドカラーに調整
3. プレースホルダー（`[PLACEHOLDER]`）を実際のコンテンツに置換
4. 写真パス、ロゴパス、QRコードパスを設定
5. 各スライドにスピーカーノートを追加（`<aside class="notes">` 形式）
6. 絶対ルール6項目を遵守

**CSS必須設定:**

```css
.reveal .slides section {
  width: 1920px;
  height: 1080px;
}
```

**Reveal.js基本構造:**

```html
<div class="reveal">
  <div class="slides">
    <section><!-- スライド1 --></section>
    <section><!-- スライド2 --></section>
    <!-- ... -->
  </div>
</div>
```

### Step 5: プレビュー検証

HTML実装後、PDF生成前に以下を必ず確認する。

**チェックリスト:**

- [ ] 各スライドが1メッセージか（複数トピック詰め込みなし）
- [ ] タイトルが40pt以上か
- [ ] 本文が18pt以上か
- [ ] 絵文字が使われていないか（全てSVGアイコンか）
- [ ] 余白が十分か（詰め込み感がないか、面積20%以上が空白）
- [ ] 画像が高解像度か（ぼやけていないか）
- [ ] 全スライドで色の統一感があるか
- [ ] スピーカーノートが記載されているか
- [ ] アスペクト比が16:9で統一されているか
- [ ] コントラスト比が4.5:1以上か（特に投影環境では重要）
- [ ] 箇条書きが5項目以内か
- [ ] 各項目が10語以内か

### Step 6: PDF生成（高画質）

```bash
# Puppeteer + Ghostscript（高画質）
node scripts/slides_to_pdf.js presentation.html output.pdf
```

**高画質設定（slides_to_pdf.js に設定済み）:**

- Puppeteer: `deviceScaleFactor: 2`（2倍解像度レンダリング）
- `?print-pdf` モードで全スライドをPDF化
- Ghostscript: `-dPDFSETTINGS=/prepress`（最高品質プリセット）
- 画像解像度: 600dpi
- フォントアウトライン化

**PDF出力時の注意:**

- Reveal.jsの `?print-pdf` パラメータで印刷レイアウトに切り替え
- アニメーションはPDFでは無効（全要素が表示された状態で出力）
- 各スライドが1ページとして出力される

## 9種のスライドタイプ

構成案作成時に、各スライドを以下の9種から選択する。

| # | タイプ名 | 用途 |
| --- | --- | --- |
| 1 | タイトルスライド | メインタイトル + サブタイトル + ロゴ |
| 2 | セクション区切り | 大きなテキスト1つ + アクセントカラー背景 |
| 3 | テキスト+画像 | 左右分割 or 上下分割 |
| 4 | フルブリード画像 | 全面画像 + オーバーレイテキスト |
| 5 | 数字強調 | 大きな数字（Inter font）+ 簡潔な説明 |
| 6 | 比較/2カラム | 左右対比 |
| 7 | フロー/ステップ | 横並び3-5ステップ |
| 8 | 引用/キーメッセージ | 大きなテキスト中央配置 |
| 9 | CTA/クロージング | 行動喚起 + 連絡先 + QRコード |

各タイプの詳細なレイアウト・CSS構造は `references/slide-design-system.md` を参照。

## チラシデザインからの転用

既存のチラシ素材をスライドに転用する場合の指針:

- **情報密度を下げる**: チラシの1面分の情報を3-5枚のスライドに分割する
- **フォントサイズを上げる**: チラシの8-10ptを18-32ptに拡大
- **余白を増やす**: チラシの詰め込みレイアウトから余白重視に変換
- **ビジュアルを主役に**: チラシでは補助的だった画像をスライドでは主役に
- **配色は引き継ぐ**: ブランドカラー、アクセントカラーは統一

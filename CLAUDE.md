# flyer-design

日本語チラシ・スライドのデザイン・作成・PDF出力スキル。制作実績の蓄積によりスキルを継続的に改善する。

## プロジェクトの目的

このプロジェクトは「チラシ & スライド作成 + スキル無限改善システム」として機能する:

1. **チラシ制作** -- chirashi-design スキルのワークフローで高品質なチラシを作成する
2. **スライド制作** -- slide-design スキルのワークフローで高品質なプレゼンスライドを作成する
3. **事例蓄積** -- 完成物を `output/flyer-output/` or `output/slide-output/` に保存し、制作ノウハウを蓄積する
4. **スキル改善** -- 制作で得た知見をリファレンス・テンプレートにフィードバックし、次回の品質を底上げする
5. **相互変換** -- 将来的に、チラシ内容→スライド、スライド内容→チラシの変換も視野に入れる

## 運用サイクル

```
制作依頼（チラシ or スライド）
    ↓
該当スキルのワークフロー実行（要件→哲学→構成→素材→実装→検証→PDF）
    ↓
output/flyer-output/ or output/slide-output/ に成果物保存
    ↓
振り返り: 何がうまくいったか、何を改善すべきか
    ↓
references/ や assets/ を更新
    ↓
次の制作で活用（性能向上ループ）
```

## 制作実績と学び

### tokumei-ai-dojo（特命AI集中道場）

- **チラシ**: ダーク x 片面 A4
- **スライド**: ダーク x 16:9 x 13枚（Reveal.js）
- **チラシでの学び**:
  - LP構造（カードUI、border-radius+shadow）はチラシに不適 → 色面ゾーンで分離する
  - 同じ見た目のブロック3つ並べはNG → 各セクションの視覚を変える（数字主役/テキスト主役/混合）
  - 1ブロックだけ背景色を変えると浮く → 薄い区切り線（0.3mm）で統一感を保つ
  - 数字は突出させる（周囲の2-4倍サイズ + Inter font）
  - 上下左右の中央揃えで視覚バランスを徹底する
  - PDF品質: deviceScaleFactor:2, /prepress, 600dpi, ダウンサンプル無効
- **スライドでの学び**:
  - チラシの高密度情報をスライドに変換する際、1メッセージ/1枚に分解する
  - チラシと同じブランドカラーを使うことで統一感が生まれる
  - 数字強調（2:1、3ヶ月、¥500,000）はスライドでも非常に効果的
  - 金線イラスト（flow-*.png）はスライドのフロー図にそのまま流用できる

### nara-ws-flyer（奈良WSフライヤー）

- **種別**: ライト系フライヤー
- **注意**: LP寄りのデザインのため、チラシの参考としては使わない

## 方針

- **チラシらしさを最優先**: Web/LP思考を排除する。色面・ゾーン・高密度がチラシの基本
- **スライドはシンプルに**: 1スライド1メッセージ。余白は武器。ビジュアル重視
- **チラシとスライドは別物**: 同じ内容でも、メディアに合わせてレイアウトを根本的に変える
- **事例は正直に評価する**: うまくいかなかった点も記録し、次回の改善材料にする
- **テンプレートは実績ベースで更新する**: 良かったパターンをテンプレートに還元する
- **AI画像生成は補助手段**: ユーザー提供写真を最優先。AI生成はユーザー確認後にのみ使用する
